关于CSS的@ AT规则的总结
04 August 2017
CSS

我要说at-rules或者AT规则，可能很多人没听过，但是大家一定都见过在CSS中字符@加一些关键词的用法，这就是at-rules。这个规则数量也不多，我在这边总结一下。

● @charset

定义被样式表使用的字符集，如下所示

@charset "utf-8";

但是这个的实际功能并不大，因为现在html页面都会定义页面的字符格式，也会覆盖CSS文件。

● @import

告诉CSS引擎包含外部的CSS样式表

@import 'test.css';

这个跟html页面的import导入样式表差不多，不过也是实际作用不大，了解知道就好。

● @namespace

告诉CSS引擎所有的内容都必须考虑使用XML命名空间前缀

这个属性我基本没有接触过，对于在XHTML上应该算是特别有用。

嵌套at-rules

下面这些是嵌套类型的AT规则，带有{}

● @document

简而言之，这个规则就是对于调用这个CSS的域名有所限制，不同的域名来执行不同的CSS样式。

这个规则属于CSS4，所以目前的浏览器支持很差，只有FireFox浏览器支持，而且还需要加前缀。过几年可以再看看。

● @keyframes

这个规则应该大家都熟悉，这是CSS3动画关键帧，用来定义动画轨迹或变化情况。

@keyframes fadeIn { 0% { opacity: 0; } 100% { opacity: 1; } }

● @media

又是一个非常非常常用的规则，媒介查询。什么响应式都是靠这个来做的。不细讲，我就大概介绍一下。

@media all and (min-width: 1280px) { /* 宽度大于1280干什么... */ } @media (-webkit-min-device-pixel-ratio: 1.5), (min-resolution: 2dppx) { /* 如果是Retina屏幕干什么... */ }

● @page

这个规则用在打印文档时候修改一些CSS属性。而且只能改变部分跟打印相关的CSS属性。

@page :first { margin: 1in; }

● @supports

这个规则是我感觉平时用得人相对少，但是很有用的一个规则。查看当前浏览器是否支持某个CSS属性。不过IE浏览器对于这个属性不支持，要用的时候需要注意一下。

@supports (display: flex) { .module { display: flex; } }
